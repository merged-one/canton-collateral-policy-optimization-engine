# Quickstart LocalNet Foundation

## Purpose

This directory holds the repo-owned foundation for a Quickstart-based LocalNet while keeping the Canton Collateral Control Plane close to upstream CN Quickstart workflows.

The implemented layer in this prompt is intentionally narrow:

- pin one upstream CN Quickstart commit
- bootstrap a detached upstream checkout under `.runtime/localnet/`
- write a repo-owned `.env.local` overlay from a documented profile
- validate the composed LocalNet configuration with upstream `make check-docker` plus `docker compose ... config`

It does not yet:

- build or start the upstream stack automatically
- upload the Control Plane DAR into Quickstart
- allocate Control-Plane-specific parties or seed confidential collateral data
- integrate a live token-standard-style asset application, custodian, venue, or financing app

## Upstream Pin

`infra/quickstart/overlay/upstream-pin.env` records the current LocalNet pin:

- CN Quickstart repo: `https://github.com/digital-asset/cn-quickstart.git`
- pinned on: `2026-03-28`
- pinned commit: `fe56d460af650b71b8e20098b3e76693397a8bf9`
- upstream Quickstart subdirectory: `quickstart/`
- upstream Quickstart `.env` values observed at that commit: `DAML_RUNTIME_VERSION=3.4.10`, `SPLICE_VERSION=0.5.3`, `JAVA_VERSION=21-jdk`

This pin is separate from the repository's current Daml SDK `2.10.4` workflow package. That version bridge is the main reason this foundation stops at checkout, configuration, and compose preflight instead of claiming full package deployment.

## Overlay Profiles

- `lean`: writes `.env.local` with `OBSERVABILITY_ENABLED=false`, `AUTH_MODE=oauth2`, and `TEST_MODE=off`
- `faithful`: writes `.env.local` with upstream-style defaults, including observability

Both profiles require a `PARTY_HINT` value shaped like `ORG-ROLE-N`. The default used by `make localnet-bootstrap` is `canton-collateral-1`.

## Reproducible Commands

From the repository root:

```sh
make localnet-bootstrap
make localnet-smoke
```

`make localnet-bootstrap` stages the pinned upstream checkout and writes `quickstart/.env.local`. `make localnet-smoke` reuses the upstream `make check-docker` target, validates the fully composed Docker configuration, and probes validator readiness if the stack is already running.

Optional overrides:

```sh
make localnet-bootstrap LOCALNET_PROFILE=faithful LOCALNET_PARTY_HINT=canton-demo-1
make localnet-smoke LOCALNET_WORKDIR=/absolute/path/to/cn-quickstart
```

## Upstream-Preserving Next Step

After `make localnet-bootstrap`, the next operator step stays with upstream Quickstart commands:

```sh
cd .runtime/localnet/cn-quickstart/quickstart
make check-docker
make build
make start
make status
```

That sequence is documented here as the intended LocalNet startup path, but it is not executed automatically by the Control Plane repo in this prompt.

## Assumptions, Mocked Surfaces, And Deferred Work

| Area | Current State | Why |
| --- | --- | --- |
| Quickstart source | real pinned upstream checkout | avoid repo forks and keep delta reviewable |
| `.env.local` overlay | real and generated by repo script | smallest practical extension point supported by upstream workflow |
| Docker compose stack validation | real `docker compose ... config` smoke | proves the pinned stack can be composed before a full build or start |
| LocalNet container startup | deferred to upstream `make build` and `make start` | avoids claiming runtime success without resolving version and package-deployment gaps |
| Control Plane DAR deployment | deferred | repo Daml SDK `2.10.4` does not yet line up with upstream Quickstart runtime `3.4.10` |
| Token-standard-style asset integration | deferred | adapter contract and settlement callback shapes still need to be pinned |
